Programacion Orientada a Objetos (POO)

Temas a ser evaluados

¿Que es la programacion orientada a objetos y para que sirve?

Es un paradigma que se basa en el concepto de "objetos". Estos objetos interactúan entre sí a través de mensajes para realizar tareas y lograr los objetivos del programa. No es un lenguaje.

Sirve para organizar y estructurar el código de manera más eficiente, facilitando el desarrollo, mantenimiento y reutilización de software.

¿Que es una clase?

Es una plantilla o un "molde" que define las características comunes y el comportamiento de un conjunto de objetos. Es decir sus atributos y metodos.

Tipos de clases
Clase concreta

Estas son las clases que pueden ser instanciadas directamente. Pueden tener atributos y métodos concretos (con implementación), y pueden heredar de clases abstractas o de otras clases concretas.

Clase abstracta

Este tipo de clase no puede ser instanciada por si sola, sino que sirve como plantilla para otras clases. Proporciona metodos(concretos o abstractos) y/o propiedades que deben ser implementadas por las clases hijas.

Se utiliza cuando deseas proporcionar una implementacion base, comun para varias subclases, pero tambien deseas definir ciertos metodos que las subclases deben implementar de manera especifica.

Caracteristicas

Obligan a la herencia.

Se representan con letras italicas.

No permite instancia o creacion de objetos.

Permite:

  • Heredar de otras clases o Interfaces.
  • Declarar variables y constantes.
  • Implementacion de metodos y propiedades.

Si tiene miembros abstractos, deben ser implementados en las clases derivadas.

¿Que es un objeto?

Es una instancia de una clase

Atributos

Se refieren a las características o propiedades que describen el estado de ese objeto.

Herencia

Es la propiedad de un objeto de tener todas las caracteristicas de la clase de la que proviene. Y no solo un objeto hereda de una clase, sino que una clase tambien puede heredar de otra.

Abstraccion

Se refiere a quitar las propiedades y acciones de un objeto para dejar solo aquellas que sean importantes. Dado que diferentes tipos de problemas requieren distinta cantidad de informacion.

Polimorfismo

Es la capacidad que tienen los objetos de una clase, en ofrecer respuesta distinta e independiente, es decir la posibilidad que una entidad tome muchas formas, mediante "reescribir" el accionar de las funciones para cada clase hija.

Permite referirse a objetos de clases diferentes mediante el mismo elemento de programa y realizar la misma operacion de diferente forma segun sea el objeto que se referencia en ese momento.

Encapsulamiento

Cuando un objeto trae consigo su funcionalidad, esta ultima se oculta.

Tipos de encapsulamiento
Publico

Todos pueden acceder a los datos o metodos de una clase que se definen con este nivel.

Protegido

No son de acceso publico, solamente accesibles dentro de su clase y subclases.

Privado

En este nivel se pueden declarar miembros accesibles solo para la propia clase.

¿Que es un metodo?

Son bloques de codigo que realizan una tarea especifica dentro de una clase. Estos pueden tener parametros de entrada y pueden devolver un valor como resultado.

Caracteristicas.
Nombre

Cada metodo tiene un nombre que lo identifica dentro de la clase.

Parametros

Los metodos pueden aceptar cero o mas parametros como entrada, que son variables utilizadas dentro del metodo para realizar operaciones.

Tipo de retorno

Algunos metodos pueden devolver un valor como resultado de su ejecucion.

Modificadores de acceso

Los metodos pueden tener modificadores de acceso que controlan su visibilidad desde otras clases(publicos, protegidos o privados).

Cuerpo del metodo

Contiene las instrucciones que se ejecutan cuando se llama al metodo.

¿Que es un metodo constructor?

Son metodos especiales que se utilizan para inicializar objetos de una clase.

Caracteristicas
Nombre

Tiene que ser el mismo nombre que la clase.

Parametros

Pueden aceptar parametros que utilizan para inicializar el estado inicial del objeto.

Inicializacion

Se utilizan para inicializar variables de instancia y realizar otras operaciones necesarias para preparar al objeto para su uso.

Llamada automatica

Se llaman automaticamente cuando se crea un nuevo objeto utilizando el operador "new".

¿Que es la sobrecarga de metodos?

Permite definir multiples metodos con el mismo nombre en una clase, pero con diferente lista de parametros.

Caracteristicas
Mismo nombre

Tienen el mismo nombre que la clase

Diferentes parametros

Cada metodo sobrecargado tiene una lista de parametros diferente, ya sea en cantidad, tipo o ambos.

Diferentes firmas

La "firma" incluye su nombre y la lista de parametros.

Compilador resuelve la llamada

El compilador determina cual de los metodos con el mismo nombre se debe ejecutar segun el numero y tipo de argumentos pasados en la llamada.

Mensajes

Es una solicitud para que un objeto realice una accion o responda una consulta.

Asociaciones

Representan las relaciones entre objetos. Se establecen mediante atributos que hacen referencia a otros objetos.

De uno a uno

Un objeto esta asociado con exactamente un objeto.

De uno a muchos

Un objeto esta asociado con uno o mas objetos.

Muchos a muchos

Varios objetos estan asociados con varios objetos.

Otros tipos de asociaciones

Agregacion

En la agregación, una clase tiene una referencia a otra clase pero las instancias de la clase secundaria pueden existir de manera independiente de la clase principal.

Caracteristicas
Independencia del ciclo de vida

Los objetos agregados pueden existir de manera independiente del objeto contenedor.

Multiplicidad

Un objeto contenedor puede contener uno o mas objetos agregados.

Reutilizacion

Los objetos agregados pueden ser compartidos por varios objetos contenedores.

Composicion

Una clase está formada por una o más instancias de otra clase (componentes) y no pueden existir de manera independiente.

Interfaces

Define un conjunto de metodos que deben ser implementados por las clases que lo utilicen. Esta solo puede obtener la firma de los metodos pero no su implementacion. En la interfaz no se pueden definir metodos concretos ni campos, solo declarar los metodos que deben ser implementados por cualquier clase que implemente la interfaz.

Se utiliza cuando deseas definir un contrato o un conjunto de comportamientos que deben ser cumplidos por diferentes clases.
